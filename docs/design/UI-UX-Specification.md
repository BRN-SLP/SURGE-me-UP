```mermaid
    E --> F[Type CONFIRM]
    F --> G[Call markAsCompromised]
    G --> H[Wallet immediately blocked]
    H --> I{Was it Primary?}
    I -->|Yes| J[Alert: Set new Primary]
    I -->|No| K[Show pending status]
```

### 8.4 Cancel Compromise Flow

```mermaid
flowchart TD
    A[Another linked wallet] --> B[View Pending section]
    B --> C[Click Cancel Compromise]
    C --> D[Confirm in modal]
    D --> E[Call cancelCompromise]
    E --> F[Wallet returns to Active]
```

### 8.5 Finalize Compromise & Claim Badges

```mermaid
flowchart TD
    A[30 days pass] --> B[finalizeCompromise called]
    B --> C[Wallet status: COMPROMISED]
    C --> D{Primary was set?}
    D -->|No| E[Identity SUSPENDED]
    D -->|Yes| F[Identity remains Active]
    F --> G[Heritage badges available]
    G --> H[User goes to /identity/badges]
    H --> I[Select destination wallet]
    I --> J[Claim badges]
    J --> K[Badges minted]
```

### 8.6 Change Primary Wallet

```mermaid
flowchart TD
    A[User on /identity/manage] --> B{Cooldown active?}
    B -->|Yes| C[Button disabled, show countdown]
    B -->|No| D[Select new Primary from dropdown]
    D --> E[Click Change Primary]
    E --> F[Call setPrimaryWallet]
    F --> G[Update header & API]
    G --> H{Was identity suspended?}
    H -->|Yes| I[Identity unsuspended]
    H -->|No| J[Show success]
```
